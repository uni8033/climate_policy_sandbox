1. 选题缘起：为什么是这个方向
项目缘起
  • 本课程前几阶段持续强调 AI 工具提升生产力：综述生成、课堂辩论、AI coding、工具链协同。
  • 因此，结课项目不只要回答一个气候问题，也要体现 AI 如何帮助我们更快建模、更快推演、更快表达。
  • 我们希望把课堂里的“AI 辅助辩论”升级成一个可复用、可展示、可迭代的气候政策博弈沙盘。
我们的判断： 这个方向同时满足课程风格、气候议题相关性和系统实现空间。

项目目标

不是替代真实谈判，而是用低成本、可交互、可解释的原型，快速暴露政策阻力与妥协路径。

本页为项目动机与课程契合度说明，不涉及外部事实性数据。

2. 现实背景：为什么气候政策天然适合做多方博弈
现实背景
三个现实约束： 气候治理同时涉及温控目标、公平原则、资金安排、能源转型与国家发展权，很难由单一主体单边决定。
  • 《巴黎协定》第 2 条同时要求温控、适应与资金流转型，并强调公平以及“共同但有区别的责任”。
  • IPCC 2023 综合报告指出，人类活动已导致全球地表温度较 1850—1900 年高出 1.1°C。
  • IPCC 还指出，现有 NDC 轨迹使本世纪超过 1.5°C 的可能性较大。
  • COP28 全球盘点成果首次写入 “transition away from fossil fuels”，显示现实谈判中的利益冲突仍然尖锐。
结论：气候政策不是单一技术选择，而是同时牵涉国家发展、能源安全、国际公平、资金支持与产业转型的多方博弈问题。
来源：UN《Paris Agreement》Art.2；IPCC《Climate Change 2023: Synthesis Report》；UNFCCC《COP28 Agreement Signals “Beginning of the End” of the Fossil Fuel Era》。

3. Research Question：一句话回答我们到底想解决什么
科学问题
如何构建一个结合政策文本、国家发展指标与气候脆弱性信息的多智能体推演沙盘，用于模拟一项气候政策在不同利益相关方之间的博弈过程，并识别其阻力来源与可行妥协路径？

问题具体

聚焦单条政策提案，而不是开放式聊天。

科学意义

关注政策设计如何受多方利益约束。

预期输出

识别阻力来源、联盟变化与妥协条件。

本页为一句话研究问题定义。

4. Why Important：为什么这个问题重要
问题重要性
  • 现实上重要： 气候政策推进不是单纯寻找技术最优解，而是同时面对公平、融资、能源安全与产业转型等多重约束。
  • 科学上重要： 它连接了气候治理、Agent-Based Modeling 与大语言模型角色扮演，是一个具有方法论价值的交叉问题。
  • 教学上重要： 我们希望把课堂辩论升级成可重复推演、可比较结果的系统，而不是一次性发言展示。
我们想补上的空缺： 在真实谈判过于复杂、传统模型门槛较高的情况下，做一个低成本但有规则约束的课堂推演原型。
来源：UN《Paris Agreement》Art.2；IPCC《Climate Change 2023: Synthesis Report》；Castro et al., 2020。

5. Data：数据来源、时间范围与尺度设计
数据方案

数据层一：政策文本

  • 来源： UNFCCC NDC Registry
  • 时间 / 空间范围： 2015 至今；全球缔约方 / 国家
  • 分辨率： 国家 - 政策文本版本
  • 是否已获取： 已确认可访问，待批量整理

代表什么： 国家公开承诺了什么政策方向与目标。
在系统中的作用： 约束 Agent 的政策表态，使其发言更接近真实承诺框架。

数据层二：国家指标

  • 来源： World Bank 指标数据库
  • 时间 / 空间范围： 以 1995 - 2024 为主；全球国家 / 经济体
  • 分辨率： 年度国家尺度
  • 是否已获取： 已确认可下载，待筛选变量

代表什么： 国家排放、发展与能源等现实条件。
在系统中的作用： 解释某类角色为什么支持或反对某项政策。

数据层三：脆弱性与适应能力

  • 来源： ND-GAIN Country Index
  • 时间 / 空间范围： 1995 至今；全球国家
  • 分辨率： 年度国家尺度
  • 是否已获取： 已确认下载入口，待与前两类数据对齐

代表什么： 国家面对气候风险时的脆弱程度与应对能力。
在系统中的作用： 帮助判断哪些角色更强调损失、适应与资金支持。

一句话理解： 这三层数据分别对应“承诺是什么”“现实条件如何”“气候风险有多高”。
来源：UNFCCC NDC Registry；World Bank 指标 EN.GHG.CO2.PC.CE.AR5；ND-GAIN Country Index。

6. Data Progress：当前进展与潜在数据风险
进展与风险

当前进展

  • 已确认三类核心数据均来自公开可查来源
  • 已初步确定最小分析单位为“国家 - 年份 + 政策文本版本”
  • 第一版原型聚焦国家尺度，暂不做城市或栅格尺度

需要尽早发现的风险

  • 可获得性： 部分文本为 PDF 且多语言，批量清洗成本较高
  • 尺度匹配： NDC 常按政策周期更新，而指标数据多为年度
  • 质量问题： 不同国家覆盖年份与缺失情况并不一致
当前判断： 数据并非拿不到，主要挑战是文本数据与数值数据的尺度对齐。因此第一版系统限定在国家尺度，优先验证方法是否跑通。
本页对应老师提出的“尽早发现数据不可获得、尺度不匹配、质量不足”要求。

7. 与已有研究的关系：为什么不是凭空想象
研究基础
  • 传统上，这类问题可放在 Agent-Based Modeling, ABM 框架下研究。
  • Castro 等在 WIREs Climate Change 的综述回顾了 61 项 气候—能源政策 ABM 研究。
  • RICE-N / AI for Global Climate Cooperation 说明，AI 与多智能体方法已经被用于模拟气候谈判、协议与长期合作。
我们的定位： 不替代成熟模型，而是利用 LLM 降低原型开发门槛，把复杂谈判逻辑做成适合课堂展示的轻量化可视化推演工具。
来源：Castro et al., A review of agent-based modeling of climate-energy policy, 2020；Zhang et al., arXiv:2208.07004, 2022。

8. 系统总体架构：从政策输入到可行性报告
系统架构
整体流程： 政策输入层 → 角色初始化层 → 多轮博弈引擎 → 观察员总结层 → 报告与可视化输出层
  • 政策输入层： 输入一条气候政策提案和可选背景条件。
  • 角色初始化层： 为各 Agent 注入立场、底线、筹码和约束。
  • 博弈引擎： 控制轮次、记录让步、判断冲突升级或缓和。
  • 观察员总结层： 提炼核心争点、共识点与失败原因。
  • 输出层： 给出对话流、评分、修改建议与简报摘要。
  • 前端层： 通过 Web 页面进行输入、展示和交互。
本页为本组系统架构设计。

9. 角色设计：把利益冲突显式写进 Agent
角色设计
角色 核心立场
发展中国家代表 强调发展权、能源安全、减贫优先，主张共同但有区别的责任。
发达国家联盟代表 支持更快减排与更强约束，倾向推动技术标准、碳边境措施。
传统能源 / 重工业代表 强调就业、资产搁浅与转型成本，更偏好渐进转型和过渡技术。
小岛国 / 环保 NGO 代表 强调气候脆弱性、损失与损害、气候正义，要求更强减排与资金支持。
AI 秘书长 / 观察员 不参与站队，只负责记录争点、识别共识、生成总结报告。
后续可扩展角色：国际金融机构、技术供应方、跨国企业、国内监管者、媒体 / 公众代理。
角色关键词来自真实气候治理议题；具体 Prompt 设计为本组方案。

10. 互动规则：让博弈像博弈，而不是轮流发言
互动规则

Agent 内部状态

  • 政策偏好
  • 不可退让底线
  • 可交换筹码
  • 当前谈判温度

交互流程

  • 立场陈述
  • 反驳与让步交换
  • 最终表态
  • 观察员归纳
  • 如果政策触碰底线，Agent 会进入强烈反对；若获得技术转移、过渡期或资金承诺，其立场可松动。
  • 观察员不判断“谁对谁错”，只判断冲突来自哪里、哪些条件最能改变局势。
本页为本组规则设计。

11. 深入机制设计：它不只是多轮对话
机制设计

机制一：证据约束

  • 为每类角色绑定议题清单与公开政策依据
  • 发言优先围绕减排、公平、融资、技术等真实议题展开
  • 观察员标记“立场依据”与“争议来源”

机制二：谈判状态机

  • 记录每轮发言后的立场变化、联盟靠拢与否决风险
  • 若触碰底线则进入强烈反对，若补偿到位则进入条件性支持
  • 支持“修订政策文本后再次博弈”的迭代流程

机制三：多情景推演

同一条政策可放入不同外部条件下重新推演，例如能源价格上行、极端气候灾害加剧、国际气候资金增加，从而比较同一政策在不同环境中的阻力变化。

亮点： 从“角色聊天”升级到“有证据约束、有谈判状态机、有情景切换、有政策迭代”的半结构化政策推演系统。
本页为项目深入机制设计。

12. 演示样例：一条政策如何被沙盘推演
演示样例
示例输入： “2030 年前停止新建无减排措施煤电，并对高碳产品征收更强碳边境调节措施。”
  • 发展中国家代表：是否影响能源安全与工业化进程？是否有资金与技术支持？
  • 发达国家联盟：支持方向，但希望加入更明确时间表与核查机制。
  • 传统能源与重工业：要求保留 CCS、过渡期和就业缓冲。
  • 小岛国 / 环保 NGO：可能认为力度仍然不足，并追问损失与损害资金安排。
预期价值：一条政策不仅得到“赞成 / 反对”，还会得到“为什么”“卡在哪里”以及“怎样改更可能通过”。
本页政策文本为课堂演示示例，不是现实谈判结果陈述。

13. 输出形式与评估方式
输出与评估

最终输出

  • 对话流：展示多轮发言与立场变化
  • 冲突地图：归纳争议集中在哪些议题
  • 联盟变化：展示潜在支持方、摇摆方与否决方
  • 三项评分：激进程度、落地阻力、公平性
  • 修改建议：生成更易获得共识的修订版本

评估方式

  • 结构上：输出是否稳定、可解释
  • 内容上：论证是否符合角色设定
  • 工程上：是否能在几分钟内生成清晰报告
  • 教学上：是否能有效辅助课堂讨论
核心价值： 这个系统不是简单展示聊天记录，而是把讨论沉淀成一个可分析、可比较、可复盘的课堂工具。
本页为输出与评估设计。

14. 技术路线：为什么这个原型这学期做得出来
技术路线
  • 数据层： NDC 政策文本 + 国家年度指标 + ND-GAIN 脆弱性 / 适应能力指标。
  • 框架层： 采用多智能体对话组织方式，参考 AutoGen 的多 Agent 协作思路。
  • 应用层： 前端优先选 Streamlit，适合快速搭建交互式 Python Web App。
  • 实现层： 角色模板 + 议题约束 + 状态变量 + 轮次控制 + 结构化输出。
  • 增强层： 外部条件注入、政策文本修订、情景重跑与结果对比。
  • 演示层： 聊天室视图 + 摘要卡片 + 联盟变化 + 评分卡片 + 报告导出。

最小可用版本

1 个输入框 + 4 个核心角色 + 1 个观察员 + 1 份自动报告。

增强版本

增加政策库、证据约束、联盟变化、引用标注、图表可视化与多情景对比。

来源：Wu et al., AutoGen, arXiv:2308.08155；Streamlit Docs, Get started with Streamlit。

16. 项目边界与真实性承诺
项目边界
  • 这个系统是教学与探索型决策支持工具，不是现实政策预测机器。
  • 它擅长暴露利益冲突、辅助课堂讨论，不擅长替代真实国家立场、精确量化排放后果。
  • 为避免“AI 编故事”，所有外部事实应来自公开可查来源；角色观点应与其设定及现实议题框架一致。
  • 后续可加入政策文件检索、发言出处标注与人工复核机制，进一步提升可追溯性。
本组结论： 这是一个既贴合课程风格、又能连接气候治理真实问题、并且具备明确工程落地路径的结课项目选题。
本页为项目边界声明。
