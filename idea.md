---

### 一、 核心科学/方法论问题

在汇报时，不要一上来就说“我们要写一个代码”，要先抛出你们要解决的**科学/方法论核心问题**：

> **“如何利用大语言模型的角色扮演（Role-playing）与多智能体（Multi-Agent）交互机制，低成本、高动态地模拟全球气候政策制定中的多方博弈，并预测政策推行的潜在阻力和共识路径？”**

---

### 二、 沙盘系统架构设计（你们的 Tool 怎么运作）

你们要做的不是一个简单的聊天机器人，而是一个“气候政策推演沙盘”。核心流程可以用一个经典的多智能体流水线（Pipeline）来设计：

1. **环境输入（Environment）**：用户（或老师）输入一项具体的政策提案。例如：*“全球在 2030 年前全面禁止新建一切煤电厂，并对高碳排国家征收 20% 的碳边境税。”*
2. **多角色智能体（Agents）**：系统自动激活 4-5 个具有完全不同利益立场的 Agent。
3. **辩论与博弈层（Debate & Negotiation）**：Agent 之间根据设定的规则进行**多轮交锋**。
4. **共识与评估输出（Output）**：由一个中立的“AI 秘书长/观察员”总结辩论过程，输出一份《政策可行性报告》。

---

### 三、 角色设定（Agent Prompting 示例）

为了让博弈有看点，Agent 的立场必须极其鲜明甚至对立。你们可以利用 Vibe Coding 快速配置以下角色：

* **Agent A：发展中国家代表**
* *立场*：强调“共同但有区别的责任”，关注经济增长、脱贫与能源安全，坚决反对一刀切的激进减排。


* **Agent B：发达国家联盟代表**
* *立场*：积极推动激进的减排时间表，提倡技术输出（但带有专利保护），倾向于使用碳关税等经济手段施压。


* **Agent C：重工业/传统能源巨头利益链**
* *立场*：强调就业率、资产搁浅风险，游说延缓转型，极力推崇碳捕集（CCS）等过渡技术而非直接关停。


* **Agent D：小岛国联盟/环保NGO代表**
* *立场*：气候变化最直接的受害者，态度最激进，要求发达国家立刻提供巨额气候补偿基金。



---

### 四、 核心功能亮点（用 Vibe Coding 怎么做）

利用代码助手（Cursor/Copilot），你们可以用 Python 快速搭建一个基于 Web 的前端界面（比如用 Streamlit 库，几行 Vibe Coding 就能搞定），实现以下炫酷功能：

* **实时对话流可视化**：在网页上像看微信群聊一样，看到代表们你来我往地针对政策进行反驳、妥协。
* **“筹码”与弹性机制**：给每个 Agent 设置初始的“经济底线”和“政治压力值”。如果某项政策逼近其底线，Agent 会触发“强烈抗议”并离席；如果对方给予技术或资金补偿，其态度会转向“部分同意”。
* **政策最终评分**：最后由大模型给该项政策打出：**激进指数、落地阻力指数、社会公平指数**，并生成最终修改建议。
