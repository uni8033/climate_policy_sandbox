选题方案汇报

CarbonWise：AI + openLCA 的场景化低碳产品决策 Agent

用 openLCA 做生命周期评估，用 大模型 把专业结果翻译成普通用户可理解、可比较、可执行的低碳选择建议。
项目类型：智能 Agent / 实用工具 首发场景：校园外卖包装比较 关键词：LCA / AI / 气候行动 / 可持续消费
我们想解决的不是“大家知不知道环保”，而是“当大家真的要做产品选择时，AI 能不能提供基于证据的低碳决策支持”。
1 个场景 从高频、低门槛、强展示性的外卖包装比较切入
2 层能力 底层严肃评估 + 上层自然语言解释与交互
3 类输出 方案比较、碳热点识别、情景模拟与建议
汇报目标：呈现选题价值、科学问题、技术路径、创新点与实施计划

02 / 11
Problem

问题定义：低碳选择很多，但证据链与可理解性不足

现实痛点

  • 很多“环保判断”停留在直觉层，比如“纸一定比塑料更环保”。
  • 真实环境影响取决于原材料、制造、运输、使用、废弃处理的全生命周期。
  • 生命周期评估专业性强，普通用户、学生组织和校园采购者很难直接使用。
  • 因此，决策难点不只是“缺数据”，更是缺一层把专业评估翻译成行动建议的接口。

项目切口

我们不做泛化的“环保聊天机器人”，而是聚焦一个足够真实、可演示、可落地的场景：

校园外卖包装比较
  • 比较对象：塑料盒、纸盒、PLA 可降解餐盒
  • 比较变量：运输距离、回收率、使用条件、功能单位
  • 输出内容：推荐方案、碳热点、情景敏感性分析、追问式解释
问题转译：从“环保口号”转向“可执行选择”。
03 / 11
Research Question

科学问题与研究假设

AI 能否将复杂的生命周期评估结果转化为非专家也能理解和采纳的低碳决策建议？

子问题 1

在具体产品选择中，哪些生命周期环节是真正的碳排热点？用户的直觉判断与生命周期结果是否一致？

子问题 2

当大模型接入结构化 LCA 结果后，是否能比一般性的环保建议提供更可信、更细化的解释？

子问题 3

在多方案对比与 what-if 情景模拟中，AI Agent 能否帮助用户更快形成基于证据的选择？

研究假设：相比纯文本型“绿色建议”，基于 openLCA 结果驱动的 Agent 更可能提供可追溯、可比较、可讨论的低碳决策支持。
这里强调：项目既有工具属性，也有明确的跨学科研究问题。
04 / 11
Method

方案逻辑：openLCA 负责“算得严肃”，AI 负责“讲得明白”

为什么选 openLCA

  • openLCA 官方将其定义为免费、专业、开源的生命周期评估软件。
  • 官网明确说明该软件适用于研究与教育场景，说明其具备较强的学术与工具属性。
  • 这使它非常适合作为本项目的“专业方法层”。
来源：openLCA 官网，GreenDelta；官网写明 “free, professional LCA software” 与 “used in many university classes worldwide”。

为什么还要叠加 AI

  • 大模型不是用来替代生命周期建模，而是用于场景理解、结果解释、方案比较、连续追问。
  • 环境决策相关研究已提示：LLM 具有潜力，但必须强调人类监督与迭代优化。
  • 因此我们采用“结构化评估 + 生成式解释”的混合架构，而不是让模型自由发挥。
来源：Cilliers et al., 2025, Impact Assessment and Project Appraisal；研究在 20 个真实 EIA 案例上测试了 3 个 GPT 版本，并强调 human oversight 与 iterative refinement。
方法论定位：不是“让 AI 直接下结论”，而是“让 AI 变成 LCA 的用户接口层”。
05 / 11
Architecture

系统架构与工作流

Step 1

用户输入场景

例如“社团活动采购 200 份外卖，比较塑料盒、纸盒、PLA 餐盒”。

Step 2

参数解析

识别功能单位、运输距离、回收假设、使用边界和对比对象。

Step 3

调用 LCA 结果

读取 openLCA 模型或预先导出的结构化生命周期结果。

Step 4

AI 解释与比较

输出碳热点、优劣比较、结论条件与风险提示。

Step 5

持续追问

支持 “如果运输更远呢”“如果回收率提高呢” 等 what-if 交互。

最小可行产品 MVP

  • 首发只做一个校园高频场景，控制边界与工作量
  • 优先实现“多方案比较 + 可解释输出 + 情景模拟”闭环
  • 前端可采用单页网页，方便本地演示与快速迭代

预期输出

  • 推荐方案与排名
  • 生命周期热点阶段
  • 结论成立的关键条件
  • 进一步降低影响的可执行建议
可视化重点：一眼看到“为什么推荐这个方案”，而不仅是一个结论。
06 / 11
Innovation

创新点与差异化

差异化 1

不是“通用环保问答”，而是场景化低碳选择，直接面向具体产品与具体比较任务。

差异化 2

不是“只用大模型生成文案”，而是把 openLCA 作为底层专业依据。

差异化 3

不是“只给结论”，而是同步展示碳热点、结论条件与反转情景。

差异化 4

不是静态报告，而是支持追问的 Agent，可处理“如果运输更远”“如果回收率提高”等问题。

差异化 5

不是宏观空谈气候变化，而是把气候行动落到微观产品决策这一真实入口。

差异化 6

最终交付是一个可交互、可演示、可继续扩展的工具原型，而不是概念海报。

项目定位：把专业生命周期分析转化为普通用户可用的低碳决策接口。

Execution

实施路径：从需求细化到原型交付

Week 1

需求与综述

  • 锁定外卖包装场景
  • 完成综述与竞品扫描
  • 定义功能单位与系统边界
Week 2

LCA 建模

  • 建立 3 个方案的简化 openLCA 模型
  • 整理影响指标和参数表
  • 确定结构化数据接口
Week 3

Agent 原型

  • 搭建前端页面
  • 实现输入-比较-解释闭环
  • 加入追问与情景模拟
Week 4

测试与打磨

  • 做小样本使用测试
  • 优化 Prompt 与图表
  • 准备现场 demo 与汇报材料

技术栈建议

生命周期层：openLCA；后端：Python / FastAPI；前端：Streamlit 或原生 HTML；Agent 层：LLM + 规则模板 + 结构化结果检索。

执行原则：先收窄场景，再做闭环，不追求“一口吃成通用平台”。
08 / 11
Evaluation

评估指标、风险与应对

评估指标

  • 功能性：能否完成多方案比较、热点识别、建议生成、情景模拟
  • 可解释性：用户是否能理解“为什么推荐这个方案”
  • 一致性：AI 解释是否与结构化 LCA 结果一致
  • 展示性：能否在汇报中完成完整 demo 闭环

主要风险与应对

  • 风险 1：LCA 建模过重
    应对：只做一个场景和三个方案
  • 风险 2：AI 输出空泛
    应对：结果必须绑定结构化指标和规则模板
  • 风险 3：看起来像普通聊天机器人
    应对：重点展示图表、对比和 what-if 模拟
  • 风险 4：结论易被质疑
    应对：保留假设条件、边界说明和来源引用
设计原则：所有结论都必须能够回溯到明确的情景假设、LCA 结果或公开来源，而不是由模型“自由编造”。
这页帮助老师相信：项目可控、可做完、也知道边界在哪里。

Outcome

预期成果与汇报展示逻辑

最终交付

  • 一个可本地演示的低碳产品决策 Agent 原型
  • 一套清晰的 LCA 场景假设与参数说明
  • 一份 AI + LCA 研究综述与案例分析
  • 一套可用于现场展示的结果图表和 demo 脚本

台上展示顺序

  • 先抛出反直觉问题：纸盒一定比塑料盒更环保吗？
  • 再讲科学问题：为什么要引入生命周期视角
  • 接着展示系统：openLCA 如何提供底层结果，AI 如何解释
  • 最后现场 demo：输入外卖场景，输出推荐方案与理由
一句话总结：我们要做的不是“更会聊天的环保助手”，而是“把专业生命周期分析转化为可执行低碳决策的智能体”。
汇报落点：兼具研究价值、产品原型可见性与后续扩展空间。
