# 明天课堂简报：项目 Pipeline 与 AI 使用

## 一、可以怎么开场

一句话版本：

> 我们现在已经不只是停留在想法层面，而是开始把“气候政策多利益相关方博弈沙盘”做成一个可运行的最小原型。

## 二、项目 Pipeline

当前版本的 pipeline 很清楚，可以直接按下面讲：

```text
政策输入
-> 解析政策属性
-> 激活四类利益相关方角色
-> 进行三轮博弈
-> 每轮后由观察员总结冲突点
-> 自动调整政策条件
-> 输出结构化政策可行性报告
```

### 每一步具体在做什么

1. `政策输入`
   - 输入一条具体政策，例如禁止新建煤电、提高碳边境约束。
2. `解析政策属性`
   - 把政策翻译成若干可计算属性，比如减排强度、碳关税、资金支持、技术转移、过渡期等。
3. `角色初始化`
   - 激活发展中国家、发达国家联盟、传统能源行业、小岛国/环保 NGO 四类角色。
4. `三轮博弈`
   - 每个角色根据自己的利益结构对政策作出回应。
5. `观察员总结`
   - 每轮识别主要冲突点和可能的修改方向。
6. `自动调整政策`
   - 如果冲突太大，就自动增加资金、技术、过渡期等妥协条件。
7. `输出报告`
   - 最后输出立场变化、阻力来源、妥协路径和简化评分。

## 三、AI 在哪里使用

### 项目推进中的 AI 使用

- AI 主要体现在：
  - 选题发散、方案比较和问题收敛
  - 角色设定、互动规则和 pipeline 设计
  - 原型代码生成、调试辅助和文档整理
  - 讲稿撰写、汇报材料组织和结果总结

### 系统原型中的 AI 使用

- 当前版本中：
  - AI 用于角色表达、自然语言表态和报告组织
  - 底层谈判结构先由`规则引擎 + 模板叙事器`保证稳定性
- 后续增强方向：
  - 接入云端大模型 API
  - 生成更自然的角色发言和更完整的观察员报告
  - 支持更多追问、多情景比较和更完整的政策简报输出

## 四、为什么不一开始就完全交给大模型

这个点明天很值得讲，因为会显得你们不是“乱用 AI”。

可以这样说：

> 我们在这个项目里并没有回避使用 AI，而是把 AI 真正用到了选题、设计、代码、结果整理和汇报准备这些环节。同时，在系统内部我们又保留了一层规则结构，保证当前原型可控、可解释、可复现。

## 五、当前已经做出的进展

- 已完成项目规划文档
- 已完成 `1.0` 原型代码框架
- 已有默认角色配置与样例政策输入
- 已能输出三轮博弈结果和结构化报告

## 七、一个 1 分钟口头版本

> 我们现在的项目 pipeline 是：先输入一条气候政策，再把它翻译成几个关键属性，比如减排强度、碳关税、资金支持和技术转移，然后激活四类利益相关方角色，让它们进行三轮博弈。每一轮后有一个观察员总结主要冲突点，并自动调整政策条件，最后输出一份结构化政策可行性报告。AI 在这个项目里其实用得比较全面，既参与了前期选题和方案设计，也参与了角色表达、结果整理和汇报材料生成；而在系统内部，底层立场变化先由规则引擎保证稳定性，后续再继续增强大模型部分。
