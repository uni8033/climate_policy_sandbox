1. 研究问题与项目定位
Research Question
核心问题： 如何构建一个结合政策文本、国家发展指标、气候脆弱性信息与多主体协商规则的推演沙盘，用于模拟气候政策在不同利益相关方之间的博弈过程，并识别阻力来源与可行妥协路径？

项目对象

聚焦单条气候政策提案，而不是开放式闲聊。

研究目标

解释谁支持、谁反对、为什么变化、怎样形成脆弱共识。

项目定位

带真实来源约束的研究型课堂原型，不是现实政策预测机器。

  • 现实意义： 气候政策同时受到减排、公平、融资、能源安全与产业转型约束。
  • 方法意义： 将 ABM 思路、规则推演与 LLM 叙事表达结合为可运行系统。
  • 课程意义： 把课堂辩论升级为可重复实验、可比较结果、可视化展示的工具。
来源：UN Paris Agreement Art.2；IPCC 2023 Synthesis Report；Castro et al., 2020。

2. 数据基础与真实性设计
Data & Traceability

政策文本层

  • 来源：UNFCCC NDC Registry
  • 对象：国家公开气候承诺
  • 作用：约束主体立场表达

国家指标层

  • 来源：World Bank Open Data
  • 对象：收入、能源、出口结构
  • 作用：生成主体偏好权重

脆弱性层

  • 来源：ND-GAIN
  • 对象：脆弱性与准备度
  • 作用：触发适应与资金诉求

当前真实国家样本

德国、美国、中国、巴西、印度、印度尼西亚、沙特阿拉伯、斐济、孟加拉国，共 9 个样本。

每个主体都绑定指标年份、政策摘要、来源目录与可靠性说明。

统一分析单位

“国家 - 年份 + 政策文本版本”

第一版刻意限定在国家尺度，避免文本周期与数值数据尺度完全错位。

真实性原则： 系统中的“主体画像”和“政策约束”尽量回到公开来源，不使用无法追溯的虚构设定。
来源：UNFCCC NDC Registry；World Bank Open Data；ND-GAIN Country Index；Climate Watch；`prototype_v1/data/source_catalog.json`。

3. 系统总体架构
System Architecture
总体架构： 政策输入层 → 属性识别层 → 主体建模层 → 六阶段博弈引擎 → 叙事与可视化输出层

前端与交互层

  • Streamlit 展示界面
  • 单次推演 / 情景实验双模式
  • 结果图表、来源目录、主体画像页

后端与推演层

  • Python 规则引擎 `climate_policy_engine.py`
  • 属性归一化、权重生成、约束检查、情景批量运行
  • 可选接入 OpenAI-compatible 大模型作为叙事层
模块 职责

Narration Layer输出可读陈述文本，支持模板模式与真实模型模式。
Visualization Layer展示可行性、公平性、阻力强度、热力矩阵等结果。

4. 核心算法 I：政策属性识别
Feature Extraction

8 个政策维度

  • 化石能源退出强度
  • 碳边境约束强度
  • 气候资金支持
  • 技术转移支持
  • 过渡期灵活性
  • CCS 过渡技术支持
  • 适应 / 损失补偿支持
  • 森林 / LULUCF 支持

识别逻辑

  • 基于关键词、否定表达和结构化规则进行识别
  • 输出每一维 `0-3` 的强度值
  • 避免完全依赖 LLM 自由判断，提高稳定性
policy_text -> normalize_attributes() -> {feature_1: 0..3, ..., feature_8: 0..3}
  • 为什么这样做： 将自然语言政策提案映射为结构化变量，后续才能进行评分、约束判断和情景批量实验。
  • 方法特点： 当前是可解释的规则识别，不是黑箱文本分类模型。
来源：`prototype_v1/climate_policy_engine.py` 中 `FEATURE_ORDER`、`FEATURE_LABELS` 与属性归一化逻辑。

5. 核心算法 II：主体权重与约束生成
Agent Modeling
关键思想： 主体不是“写死的性格设定”，而是由公开指标、NDC 摘要和条件性承诺共同生成偏好与底线。

权重生成变量

  • 人均 GDP
  • 可再生能源占比
  • 燃料出口依赖
  • ND-GAIN 脆弱性 / 准备度
  • NDC 目标类型
  • 条件性融资 / 技术表述

现实约束类型

  • min_requirements： 至少需要的资金、适应或技术安排
  • max_tolerances： 最多能接受的退出强度或碳边境约束
  • 约束触发后，主体立场显式下调
score(agent, policy) = base_bias + Σ(weight_i × feature_i) - constraint_penalty
  • 结果： 同一条政策面对不同主体，会出现结构化、可解释的支持与反对差异。
  • 优势： 比纯角色扮演更稳，比完全手写脚本更灵活。
来源：`prototype_v1/climate_policy_engine.py` 中 `_derive_country_weights()`、`_build_country_constraint_profile()` 等逻辑。

6. 核心算法 III：六阶段谈判引擎
Negotiation Engine
1立场陈述
2冲突识别
3联盟与分裂
4条件交换
5秘书处修订
6最终表决

引擎做什么

  • 控制阶段顺序与多轮更新
  • 统计主体立场变化与联盟版图
  • 根据约束命中调整得分和文本输出

为什么不是自由聊天

  • 每一步有固定目标和状态转移
  • 修订阶段允许有条件妥协，而非随意漂移
  • 最终表决收束为可比较结果而非开放对话
方法意义： 这使系统同时具备“自然语言可读性”和“过程可解释性”。
来源：`PIPELINE_STAGES` 六阶段流程设计与当前单次推演输出结构。

7. 架构特点：规则引擎 + LLM 叙事层
Hybrid Architecture

规则层负责

  • 数据加载与标准化
  • 属性识别与主体评分
  • 现实约束检查
  • 情景批量运行与指标汇总

LLM 层负责

  • 生成更自然的陈述文本
  • 总结谈判叙事与研究观察
  • 支持模板模式与真实模型模式切换
  • 兼容 OpenAI-compatible 接口
  • 设计原因： 直接让 LLM 决定结果，容易出现漂移和不可复现；完全不用 LLM，又会损失表达效果。
  • 因此： 结构化判断交给规则引擎，表述生成交给 LLM，形成混合架构。
一句话总结： 结果由规则收敛，表达由模型增强。
来源：`prototype_v1/llm_adapter.py`、`prototype_v1/climate_policy_engine.py`、当前 Streamlit 原型配置。

9. 样例结果：当前系统已经能输出什么
Example Output
样例输入： “2035 年前在加强森林保护、碳汇和土地利用治理的同时，逐步停止新建无减排措施煤电，并为高脆弱国家提供适应融资与技术转移支持。”

当前发现

  • 公平融资优先情景：可行性 77.8
  • 公平融资优先情景：公平性 80.0
  • 公平融资优先情景：阻力强度 22.2
  • 森林治理优先情景也显著改善接受度

结果含义

  • 高雄心政策本身并不足够
  • 融资、适应支持和 LULUCF 安排会显著影响共识
  • 系统已能比较哪类政策组合更可能跨过谈判门槛

展示价值

输出不再只是“谁支持谁反对”的聊天记录，而是能沉淀为研究发现、情景比较和可视化结果图。

来源：`prototype_v1` 当前样例运行结果；用于说明原型能力，不代表现实政策预测。

11. 技术栈与 AI 使用方式
Tech Stack

开发语言

Python

负责规则引擎、数据处理、实验运行。

展示层

Streamlit + Vega-Lite

负责网页界面、图表和交互展示。

AI 接口

OpenAI-compatible API

支持模板模式与真实模型模式切换。

  • AI 的作用 1： 帮助前期方案发散、原型开发与代码实现。
  • AI 的作用 2： 在系统内部承担叙事生成与解释增强，而不是完全替代结构化判断。
  • AI 的作用 3： 提升展示表达效率，包括图表、界面与 PPT 材料制作。
方法态度： 全流程使用 AI，但所有事实性内容、数据来源和模型边界都要求显式核查。

12. 项目贡献、边界与局限
Contributions & Limits

当前贡献

  • 完成可运行原型，而非停留在概念设计
  • 引入真实国家样本、公开来源和可靠性说明
  • 形成规则引擎 + LLM 的混合架构
  • 支持多情景实验与结果可视化

当前局限

  • 不是 IAM、CGE 或能源系统模型
  • 权重映射仍有经验型成分，尚未完全计量标定
  • 文本解析仍以轻量规则为主，未引入更强语义解析
  • 样本国家数量仍有限，尚未覆盖完整谈判格局
边界声明： 本项目适合做课堂研究展示、政策比较与协商结构分析，不适合被解释为真实政策效果预测器。
来源：`prototype_v1/README.md`、`research_mode_notes.md`、`real_world_data_notes.md`。

13. 结论与下一步工作
Conclusion
  • 本项目已经形成“研究问题 - 数据 - 方法 - 原型 - 实验 - 局限”相对完整的大作业闭环。
  • 技术上，核心创新不在于单独使用 LLM，而在于将规则引擎、真实国家样本与叙事生成组合成可解释系统。
  • 研究上，系统已经能比较不同政策组合的谈判接受度，而不是只展示一次性对话。

下一步 1

继续扩展真实国家与文献校准，提高权重科学性。

下一步 2

增强联盟网络图、情景雷达图与更细的文本解析能力。

下一步 3

整理为正式报告、图表包与可复现实验材料。

结论： 这个项目的价值不只在于“做了一个 demo”，而在于已经构成一个可展示、可复盘、可继续研究的气候政策推演原型。
本页为项目总结与后续工作规划。
