# Prompt 与 AI 使用说明

## 当前版本为什么没有强制接入在线大模型

当前 `1.0` 版本的目标是：

- 先稳定跑通系统闭环
- 先把角色、状态、报告结构做清楚
- 先形成一个课堂上能讲清楚的方法原型

因此，当前版本采用：

- `规则引擎` 负责立场计算
- `模板叙事器` 负责生成较自然的发言文本
- `可选 LLM 适配器` 负责后续升级

## 当前 AI 使用方式

### 方式一：Prompt 风格角色模板

每个角色都包含：

- 身份描述
- 核心利益
- 政策底线
- 偏好的补偿机制

这些内容虽然现在主要由模板表达，但本质上已经是 Prompt 设计。

### 方式二：结构化到自然语言的转换

规则引擎先给出：

- 角色得分
- 当前立场
- 支持点
- 反对点
- 谈判诉求

然后由叙事层把这些结构化信息转成课堂可读的中文发言。

### 方式三：预留云端 LLM 升级接口

`llm_adapter.py` 里预留了 `OpenAI-compatible` 的调用方式。

后续可使用：

- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`
- `OPENAI_MODEL`

把当前模板生成切换成真实大模型生成。

## 后续推荐 Prompt 框架

### Agent 系统提示模板

```text
你是一个气候政策谈判中的角色代理。
你必须坚持自己的利益立场，但可以在获得合适补偿条件时作出有限让步。
你的输出必须围绕政策文本、现实条件、公平原则和气候风险展开。
```

### 每轮用户提示模板

```text
当前政策提案如下：{policy_text}
当前政策属性如下：{policy_attributes}
你代表的角色是：{role_name}
你的角色背景是：{role_description}
请先说明你的态度，再说明支持或反对的原因，最后提出你愿意接受的修改条件。
```

### 观察员提示模板

```text
请总结本轮发言中的主要冲突点、共识点和可能的修改方向。
输出必须包含：主要阻力来源、可能的妥协条件、下一轮建议修改。
```

## 明天课堂可以怎么讲

一句话版本：

> 当前 1.0 版本里，我们先用规则引擎把谈判结构固定住，再把 AI 放在角色表达和报告生成层，这样比一开始完全让大模型自由发挥更稳、更可解释。
