# [OPEN] Debug Session: streamlit-preview-unavailable

## 1. 问题摘要

- 目标：让网页端可以直接打开并演示 `Streamlit` demo
- 当前现象：预览显示“服务不可用”
- 当前服务命令：
  - `python3 -m streamlit run streamlit_app.py --server.headless true --server.port 8502`

## 2. 当前状态

- 状态：`OPEN`
- 调试阶段：`假设与证据收集`
- 当前约束：在获得更多运行时证据前，不修改业务逻辑

## 3. 初始假设

1. `Streamlit` 服务已启动，但预览访问地址或端口不正确。
2. 服务器进程存在，但应用首屏渲染阶段报错，导致前端显示服务不可用。
3. 预览系统无法访问 `8502` 端口，而不是应用本身启动失败。
4. 当前环境变量或默认运行模式导致应用启动后立即异常。

## 4. 下一步计划

1. 检查 `Streamlit` 进程当前状态和最新输出
2. 直接请求本地 `8502` 端口收集 HTTP 证据
3. 如服务存活，再检查页面是否存在运行时异常
4. 基于证据决定是否需要最小修复

## 6. 假设验证结果

1. `Streamlit` 服务已启动，但预览访问地址或端口不正确
   - `当前最可能成立`
   - 证据：服务本身健康，但 IDE 预览返回 `ERR_ABORTED`

2. 服务器进程存在，但应用首屏渲染阶段报错

   - 证据：根路径返回 `200`，健康检查正常

3. 预览系统无法访问 `8502` 端口
   - `部分成立`
   - 证据：服务本地可达，但内嵌预览未正常打开

4. 当前环境变量或默认运行模式导致应用启动后立即异常
   - `否定`
   - 证据：服务已稳定启动，且无启动时报错

## 7. 当前结论

- 当前没有发现 `Streamlit` demo 本身的运行错误
- “服务不可用”更像是：
  - 旧预览链接失效
  - IDE 内嵌预览与 `Streamlit` 交互方式不兼容
  - 或预览层对该端口/会话的访问异常
- 本次未对业务逻辑做修复，仅补充了更适合直接演示的默认环境变量读取和说明面板
